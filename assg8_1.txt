Data Access Layer (DAL) Implementation Summary
1. Database and Tables — database.js

The database.js file initializes an SQLite database and creates four tables automatically when the server starts.

Tables Overview
Table Name	Purpose	Key Columns
users	Stores user account data	id, fullName, username, org, email, phone, dob, password, createdAt
alerts	Stores monitoring alerts	id, service, threshold, severity, status, time
notifications	Logs alert notifications	id, time, service, alert, severity, status, channel, timestamp
metrics_log	Stores system metrics	id, cpuUsage, memUsage, diskUsage, netInbound, netOutbound, recordedAt

All tables are created using CREATE TABLE IF NOT EXISTS.

2. Data Access Layer — dal.js

The dal.js file acts as the Data Access Layer, handling all database interactions. It provides structured modules with CRUD operations.

metricsDAL
logEntry()

How the DAL Architecture Works
server.js (Business Logic)
        |
        | Calls DAL methods (no SQL here)
        v
dal.js (Data Access Layer)
        |
        | Contains all SQL queries
        v
database.js → SQLite Database (verolla.db)
Key Idea
server.js never writes SQL queries directly
It only calls functions like:
userDAL.insert(...)
alertDAL.findAll()
notificationDAL.insert(...)

This ensures proper abstraction and separation of concerns.
