## Use Case «include» Relationships

The following «include» relationships represent **mandatory and reusable behaviors** in the system:

- **Sending Alerts** → «include» **Alert Generation**  
  Alerts must be generated before they can be sent.

- **Alert Acknowledgement** → «include» **Sending Alerts**  
  An alert can be acknowledged only after it has been sent.

- **Escalate Alerts** → «include» **Sending Alerts**  
  Alert escalation applies only to alerts that have already been sent.

- **Data Filtering and Searching** → «include» **Historical Data View**  
  Filtering and searching require historical data to be displayed first.

- **Alert Generation** → «include» **Metrics Management**  
  Alerts are generated by evaluating monitored metrics and thresholds.
