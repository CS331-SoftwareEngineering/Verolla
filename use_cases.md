# Use Case Descriptions  
## Monitoring & Alert Automation System

### User Registration  
This use case allows new users or administrators to create an account in the system by providing necessary details such as username, email, and password. The system validates the input information and stores the user details securely. Upon successful registration, the user is granted access based on default role settings.

### Login  
This use case enables registered users to securely access the system by entering valid login credentials. The system verifies the credentials and, upon successful authentication, grants access to system features based on the user’s assigned role.

### User Profile Management  
This use case allows users to view and update their personal information, including password, email address, and notification preferences. The system validates the updated details and applies the changes securely without affecting other users.

### Role Management  
This use case allows administrators to assign, update, or revoke user roles such as Admin or Viewer. Based on the assigned role, the system controls the user’s permissions and access to various system functionalities.
