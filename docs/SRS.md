## 1. User Requirements

- To monitor system metrics such as CPU usage, memory usage, disk usage, and application status.
- To view the real-time system health status through a dashboard.
- To receive timely alerts when system metrics exceed the configured thresholds.
- To choose notification methods for receiving alerts.
- To view detailed alert information.
- To access historical alert logs.
- To receive genuine alerts.
- To modify threshold values.
- To monitor application availability.
- To securely access the monitoring system.

## 2. Functional Requirements

- To continuously collect system performance metrics at configurable time intervals.
- To compare collected metrics against predefined threshold values.
- To detect abnormal conditions when threshold violations occur.
- To generate alerts for detected abnormal conditions.
- To send notifications to users upon alert generation.
- To log and retrieve all monitoring data and alert events.
- To display system health status in real time.
- To support configuration of monitoring thresholds.
- To restrict access to authorized users only.

## 3. Non-Functional Requirements

- To operate continuously without manual intervention.
- To generate alerts within an acceptable time after detecting an abnormal condition.
- To be reliable and minimize false alerts.
- To be available for monitoring throughout the clock.
- To provide a user interface that is easy to understand.
- To maintain and update periodically.
- To be scalable to support monitoring of multiple systems.
- To ensure accuracy in metric monitoring and alert generation.
