# II. Implementation of UI Components and User Interaction

The user interface for the monitoring system was implemented as a **web-based dashboard** using **HTML, CSS, JavaScript, and Chart.js**.

## Implemented UI Components

The following UI components were developed:

- **Dashboard interface** displaying system health metrics  
- **Metrics page** showing detailed performance charts  
- **Alerts management page** for configuring alert thresholds  
- **Notifications page** displaying alert history  
- **Settings page** for system configuration  

## Reusable UI Components

The system also includes several reusable interface components used across multiple pages:

- **Sidebar navigation menu**
- **Top navigation bar**
- **Search functionality**
- **Notification icon**
- **User profile display**

These components provide a consistent layout and improve usability across the entire monitoring dashboard.

## User Interaction Flow

1. The user logs into the monitoring system.
2. The **Dashboard** displays real-time CPU, memory, and overall system status information.
3. The user navigates to the **Metrics page** to analyze detailed performance graphs.
4. The user configures alert thresholds in the **Alerts page**.
5. When thresholds are exceeded, the system generates alerts that appear in the **Notifications page**.
6. The user can modify system preferences and configuration options in the **Settings page**.
