# I. Choice of User Interface and Justification

For the continuous monitoring and alerting system, a **Direct Manipulation Interface combined with a Menu-Based Interface** was selected.

## Direct Manipulation Interface

A **Direct Manipulation Interface** allows users to interact directly with visual elements such as graphs, cards, buttons, and controls.  
In the monitoring system, users interact with:

- Real-time **CPU, memory, and network graphs**
- **Metric cards** displaying system statistics
- **Alert configuration forms**
- **Interactive charts and tables**

This interface is suitable because monitoring systems require users to **quickly interpret system performance through visual representations of data**. The graphical dashboard allows administrators to easily identify abnormal system behavior and respond to alerts efficiently.

## Menu-Based Interface

A **Menu-Based Interface** is implemented through the **sidebar navigation menu**, which allows users to access different modules of the system. The menu includes:

- Dashboard
- Metrics
- Alerts
- Notifications
- Settings

This structure helps users navigate the system in an **organized way** and quickly access different functionalities without remembering commands.

## Justification

The combination of **Direct Manipulation** and **Menu-Based interfaces** was chosen because:

- Monitoring systems rely heavily on **visual dashboards for real-time analysis**.
- Users can interact intuitively with **graphs, alerts, and controls**.
- The menu-based navigation ensures **simple and structured access** to system features.
- It improves **usability, efficiency, and overall user experience**, which are important for system administrators managing large-scale monitoring tasks.

Therefore, this UI design is well suited for a **continuous monitoring and alerting platform**.
